# Week 3 — Python Automation, Log Forensics & System Auditing

## Objective
Build security automation skills by developing Python tools that replace manual investigation. This week focused on scripting log parsers, threat detectors, and system auditors—core capabilities used in SOC, DFIR, and security engineering roles.

## What I Worked On
- Developed Python scripts that automate port scanning, log analysis, and process auditing  
- Used lists, loops, socket connections, and defensive programming to validate network access  
- Parsed authentication logs to detect brute‑force attempts and generate structured reports  
- Monitored system processes using Python’s `subprocess` module and produced JSON‑formatted alerts  
- Practiced secure file handling, error checking, and modular script design  

## Artifacts
- 🧪 **port_check.py** — Automated port scanner that tests Port 22 across multiple hosts  
- 🔍 **brute_detector.py** — Log forensics tool that identifies brute‑force authentication attempts  
- 🛡️ **system_auditor.py** — Process monitoring script that flags unauthorized activity and outputs JSON alerts  

## Skills Demonstrated
- Python scripting for security automation and log parsing  
- Network access validation using sockets and iterative host testing  
- Threat detection through log filtering, pattern matching, and event counting  
- Process auditing using system commands and structured JSON reporting  
- Defensive programming, error handling, and secure file operations  

## Outcome
Successfully automated key security tasks by building Python tools that detect threats, validate network access, and audit system processes—demonstrating practical skills used in real‑world SOC and DFIR environments.
