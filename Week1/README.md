# Week 1 — Linux Security Foundations

## Objective
Build foundational Linux security skills by navigating the filesystem, correcting permission issues, and analyzing logs for early indicators of compromise.

## What I Worked On
- Explored the Linux filesystem to locate sensitive paths, hidden directories, and system‑level artifacts  
- Retrieved restricted information from protected locations to simulate post‑exploitation discovery  
- Identified insecure permission configurations and applied corrective hardening  
- Parsed log files using tools like `grep`, `awk`, and `sort` to detect suspicious activity  
- Performed incident cleanup by restoring proper access controls and collecting forensic evidence  

## Artifacts
- 🗂️ **discovery.txt** — Filesystem reconnaissance and extracted findings  
- 🔧 **harden.sh** — Automated permission hardening and secure configuration script  
- 📄 **threat_ips.txt** — Filtered list of suspicious IP addresses from log analysis  
- ⚠️ **final_threat_report.txt** — Forensic summary of attacker behavior and recovered evidence  

## Skills Demonstrated
- Confident terminal‑based navigation of Linux systems  
- Understanding of permission models, access control weaknesses, and secure remediation  
- Ability to extract indicators of compromise (IOCs) from log data  
- Hands‑on experience with basic incident response and forensic documentation  

## Outcome
Successfully secured a compromised Linux environment by correcting permission flaws, identifying attacker activity, and producing validated forensic artifacts.
