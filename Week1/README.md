# Week 1 — Linux Security Foundations 

## Objective
Develop core Linux security skills by practicing system navigation, permission hardening, and basic threat detection using command‑line tools.

## What I Worked On
- Navigated the Linux filesystem to identify sensitive paths and hidden artifacts  
- Retrieved restricted information from protected directories to simulate post‑exploitation discovery  
- Identified and corrected insecure permission configurations  
- Parsed and filtered log data using tools such as `grep`, `awk`, and `sort`  
- Performed incident cleanup by restoring proper access controls and collecting forensic evidence  

## Artifacts
- **discovery.txt** — Filesystem exploration and extracted findings  
- **harden.sh** — Script implementing secure file permissions and system hardening  
- **threat_ips.txt** — Filtered list of suspicious IP addresses from log analysis  
- **final_threat_report.txt** — Forensic summary of attacker activity  

## Skills Demonstrated
- Confident terminal‑based navigation of Linux systems  
- Understanding of permission models and common access control weaknesses  
- Ability to extract indicators of compromise (IOCs) from log files  
- Hands‑on experience with basic incident response and forensic documentation  

## Outcome
Successfully secured a compromised Linux environment by correcting permission flaws, identifying attacker behavior, and producing validated forensic artifacts.
